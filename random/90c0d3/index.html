Chrome book 初体验

January 7, 2021 · 1 min · laekov

不久之前听说可能要出去背锅, 然而自从 macbook 死亡之后就一直在用实验室的台式机/宿舍买的新的游戏本. 都不太适合背着去干活, 于是考虑买一台轻薄的工作笔记本.

看了一些 ryzen 4 代的轻薄本, 比如 x13, 战x, 灵耀14, 什么的. 然而因为贵所以买不起没买. 后来某天逛闲鱼的时候发现了 chromebook 很便宜, 于是一番研究之后买了个二手 hp g1 chromebook, 1k 块不到.

转身围观一下这台本子，深灰色金属壳还挺高端。最厚的地方只有 12mm，几乎满足了我对薄本的所有想象，但是拿在手里挺沉，可能是压扁了的重型本。 cpu 是六代 m5 skx, 四个物理核默认只开了两个，8 GB mem + 32 GB ssd, 多开几个 chrome 窗口应该都挺好的。最惊人的是居配了一块 3200x1800 的屏幕，观感很不错。 然而还是有一些问题的。屏幕边框宽到跑马车，毕竟是便宜本，键盘键程短到手指疼，只能轻轻打字。触摸板比较小，而且可能由于前任主人用得比较多，所以油油的，而且有点松。考虑之后买个触摸板膜。接口方面带了一个 usb a 两个 usb c包括充电，还有耳机口。考虑到现在 typec 的本子到处飞，转接头很普遍，这也还ok。这个 cpu 最高支持到 4k 屏幕，想要外接显示器也没有问题，赞。

软件方面自然是 chrome os，费了一番力来搞开开发者模式用 crounton 装了一个 debian, 和搞上 play store. 比较无语的是 play store 并不用 chrome os 里面设置的 socks5 代理，于是只好临时搞了一个二层的有隧道的网关，才成功搞开了 play store, 然后立即在里面下了一个 android 版本的 ss, 然后用它就能给 android app 供应网络了。

然后发现微信和腾讯会议都 not available in play store, 听说要开 native bridge, 于是开始研究咋用一个叫 aroc 的仓库来给 chrome os 带的 android 搞 root, 发现脚本有各种问题，会把整个 android 子系统都给搞坏，甚至还 chrome os recovery 了好几次。感到悲伤。 后来打开了小米商场，发现直接下 apk 就能安装？？？？？

然而微信和腾讯会议对 x86 android 的支持真是垃圾，根本登不进去。腾讯会议加入会议死机，共享屏幕死机。强调一遍：腾讯公司是开源软件在中国发展的最大阻碍。

然而 zoom 的国产版本瞩目用起来就毫无障碍，telegram 和 mendeley 也运行得很好。

原生软件方面主要就是一个巨大的 chrome, 甚至在可以在 chrome 里使用一个还不错的 crosh 当 terminal emulator, 按一下全屏键之后就没有快捷键冲突问题了， ssh 到服务器上就像一个普通的本子一样，好评。 电池方面这个本子不太行。没有完整测过续航，系统的剩余估计在 2h-8h 之间波动，估计 os 上省电还是做了不少工作的，但是 m5 这个 cpu 对于 chromebook 的设计可能还是太强了一点。
